# Image Toolkit 需求文档

> 一个图片处理平台，采用 Next.js（App Router，SSR）输出页面外壳，所有图片/视频处理及 AI 抠图均在浏览器本地完成，且 AI 模型资源完全本地自托管 —— **可彻底离线运行，不访问任何外部网络**。

## 1. 技术栈

| 类别     | 选型                                                                             |
| -------- | -------------------------------------------------------------------------------- |
| 框架     | Next.js 16（App Router，SSR，--webpack）+ React 18 + TypeScript                  |
| 渲染     | 服务端渲染页面外壳；交互与处理为客户端组件                                       |
| 路由     | Next.js App Router（文件路由，页面位于 `app/<route>/page.tsx`）                  |
| UI 组件  | shadcn/ui                                                                        |
| 样式     | TailwindCSS                                                                      |
| 国际化   | react-i18next（中文默认 / 英文）                                                 |
| 主题     | 明亮 / 暗黑 / 跟随系统（可切换）                                                 |
| 背景移除 | @imgly/background-removal（浏览器内 ONNX；模型与 wasm 完全本地自托管，离线可用） |
| 打包下载 | jszip + file-saver                                                               |
| 视频截帧 | HTML5 Video + Canvas                                                             |
| 图标     | lucide-react                                                                     |

> 运行：`npm run dev`（默认 http://localhost:3000）；构建：`npm run build` 后 `npm start`。
> 首次 `dev`/`build` 会执行 `scripts/copy-imgly-data.mjs`，将与库版本匹配的 AI 模型/wasm 资源下载到 `public/imgly`（已缓存则跳过）。该步骤需联网一次，之后应用运行完全离线。

## 2. 全局特性

- **主题切换**：明亮 / 暗黑 / 跟随系统，状态持久化到 localStorage。
- **多语言**：中文（默认）、英文，可切换，状态持久化。
- **隐私 / 离线**：所有图片/视频处理均在本地浏览器完成，不上传任何服务器。AI 抠图的模型与 onnxruntime wasm 由 `scripts/copy-imgly-data.mjs` 下载到 `public/imgly` 本地自托管，运行时通过 `publicPath` 指向本站 `/imgly/` 加载，**不访问任何外部 CDN，可完全离线使用**。
- **所有功能参数均为用户可配置，避免硬编码固定值。**
- **批量下载**：处理结果支持单张下载或打包为 zip 批量下载。
- **路由缓存**：功能状态提升到全局 store，切换路由后数据不丢失。
- **图片放大预览**：原图与结果图均支持点击放大（Lightbox），透明结果以棋盘格背景展示。

## 3. 功能一：图片透明底（背景移除）

### 3.1 输入

- 支持拖拽 / 点击上传，可多选（批量）。
- 支持常见格式：PNG / JPG / JPEG / WEBP。

### 3.2 抠图模式（用户可选）

1. **AI 智能抠图**
   - 自动识别主体，去除任意背景。
   - 可选模型质量档位（速度优先 / 质量优先）。
2. **指定颜色抠图（色键 Chroma Key）**
   - 取色器选择要去除的背景颜色（或在预览图上吸管取色）。
   - 可调 **容差（Tolerance）**：颜色匹配范围。
   - 可调 **边缘羽化（Edge Feather）**：边缘平滑程度。

### 3.3 输出配置（用户可选）

- 导出格式：PNG（保留透明）/ WEBP（保留透明）/ JPG（不支持透明，用预览背景色填充，透明则填白）。
- 可选预览背景色（棋盘格或指定颜色）。
- 单张下载 / 批量打包 zip 下载。

### 3.4 交互

- 处理前后对比预览（棋盘格透明背景），原图/结果图均可点击放大。
- 批量任务列表，显示每张处理状态（等待 / 处理中 / 完成 / 失败）。
- **重复抠图**：单张可在调整参数后重新抠图。
- **结果勾选**：完成的结果可勾选，支持全选/全不选，仅打包下载勾选项。

## 4. 功能二：视频截帧

### 4.1 输入

- 支持拖拽 / 点击上传视频（MP4 / WEBM / MOV 等浏览器可解码格式）。

### 4.2 取帧方式（用户可选）

1. **固定时间间隔**：每隔 N 秒截取一帧（支持到 0.001s 的小间隔）。
2. **按帧率(FPS)**：填目标帧率，自动按 1/FPS 间隔取帧，适合还原丝滑动画。
3. **指定总帧数**：在视频时长内均匀截取 M 帧（M 可配）。
4. **手动截帧**：拖动进度条到任意时间点，点击按钮截取当前帧（追加，不清空）。
5. **时间区间**（可选增强）：限定开始/结束时间，再配合上面方式。

> 截帧原理：通过设置 `video.currentTime` 精确 seek 到目标时间点，等待 `seeked` 后用 canvas 抓取该帧，按时间精确取帧（非依赖播放）。

### 4.3 输出配置（用户可选）

- 导出格式：PNG / JPG / WEBP。
- **输出分辨率**：按比例缩放输出像素尺寸（原始～10%），影响清晰度与体积，不是质量压缩。
- JPG/WEBP 质量可调（独立的"图片质量"滑块，做有损压缩）。
- 单张下载 / 批量打包 zip 下载。

### 4.4 交互

- 视频预览播放器，一次只处理一个视频，重新上传按钮为"重新添加"（替换当前视频并清空旧帧）。
- 选择"固定间隔/帧率/指定帧数"后，实时预估输出张数；数量过大时给出耗时提醒，并显示提取进度。
- **批量提取会先清空上一次的结果**；手动截帧为追加。
- 截取的帧以缩略图网格展示，可单独删除、单独下载、全选打包，点击可放大。
- **帧动画预览**：将已截取的帧按可调帧率（FPS，1–60）播放一次（不循环），到末帧停止；可重新播放。

## 5. 目录结构

```
image-toolkit/
├── app/                  # Next.js App Router（SSR 外壳与路由）
│   ├── layout.tsx        # 根布局（服务端组件）+ 主题预设脚本
│   ├── providers.tsx     # 'use client' 客户端 Providers（主题/i18n/状态/布局）
│   ├── page.tsx          # 首页重定向
│   ├── backgroundRemoval/page.tsx   # 透明底页面（'use client'）
│   └── videoFrames/page.tsx         # 视频截帧页面（'use client'）
├── src/
│   ├── components/        # 可复用 UI 组件
│   │   ├── ui/            # shadcn 基础组件
│   │   └── common/        # 业务通用组件（上传区、主题/语言切换、Lightbox 等）
│   ├── features/
│   │   ├── BackgroundRemoval/   # 透明底功能
│   │   └── VideoFrames/         # 视频截帧功能
│   ├── lib/               # 工具函数（抠图、截帧、zip 等）
│   ├── i18n/              # 国际化配置与语言包
│   └── stores/            # 全局状态（功能数据缓存，跨路由保留）
├── scripts/
│   └── copy-imgly-data.mjs  # 下载 AI 模型/wasm 到 public/imgly（离线自托管）
├── public/imgly/          # 本地 AI 资源（脚本生成，已 gitignore）
├── next.config.mjs        # Next 配置（--webpack；含 onnxruntime-web 打包处理）
└── ...
```

> 页面级组件直接位于 `app/<route>/page.tsx`（App Router 习惯用法）；业务/可复用代码放在 `src/`。

## 6. 暂不包含（后续可扩展）

- 手动画笔擦除 / 修补背景。
- ffmpeg.wasm 高级视频处理。
- 用户账户 / 云端存储。
