# Image Toolkit 需求文档

> 一个图片处理平台，采用 Next.js（App Router，SSR）输出页面外壳。图片/视频处理在浏览器本地完成；AI 抠图在浏览器内运行（ONNX/WASM），其模型权重首次使用时从线上 CDN 下载并由浏览器缓存（需联网）。

## 1. 技术栈

| 类别     | 选型                                                                             |
| -------- | -------------------------------------------------------------------------------- |
| 框架     | Next.js 16（App Router，SSR，--webpack）+ React 18 + TypeScript                  |
| 渲染     | 服务端渲染页面外壳；交互与处理为客户端组件                                       |
| 路由     | Next.js App Router（文件路由，页面位于 `src/app/<route>/page.tsx`）              |
| UI 组件  | shadcn/ui                                                                        |
| 样式     | TailwindCSS                                                                      |
| 国际化   | react-i18next（中文默认 / 英文）                                                 |
| 主题     | 明亮 / 暗黑 / 跟随系统（可切换）                                                 |
| 背景移除 | @imgly/background-removal（浏览器内 ONNX；模型权重走线上 CDN，首次按需下载缓存） |
| 打包下载 | jszip + file-saver                                                               |
| 视频截帧 | HTML5 Video + Canvas                                                             |
| 图标     | lucide-react                                                                     |

> 运行：`npm run dev`（默认 http://localhost:3000）；构建：`npm run build` 后 `npm start`。
> 代码统一位于 `src/` 下（含 `src/app` App Router 目录），符合 Next.js “src 目录”官方约定。

## 2. 全局特性

- **主题切换**：明亮 / 暗黑 / 跟随系统，状态持久化到 localStorage。
- **多语言**：中文（默认）、英文，可切换，状态持久化。
- **隐私**：图片/视频处理均在本地浏览器完成，不上传任何服务器。AI 抠图同样在浏览器内运行（不上传图片）；仅模型权重首次从线上 CDN 下载并缓存，因此 **AI 抠图需要联网**（UI 内有明确提示）。
- **所有功能参数均为用户可配置，避免硬编码固定值。**
- **批量下载**：处理结果支持单张下载或打包为 zip 批量下载。
- **路由缓存**：功能状态提升到全局 store，切换路由后数据不丢失。
- **图片放大预览**：原图与结果图均支持点击放大（Lightbox），透明结果以棋盘格背景展示。

## 3. 功能分类总览（产品视角）

> 站在产品经理视角，参考 iLoveIMG、佐糖（PicWish）、TinyPNG、做好图等同类产品的功能组织方式，按**用户意图**（而非技术实现）对功能进行分类整合。导航采用**左侧分组侧边栏**（分类 → 功能 两级结构）。

| 分类                         | 功能       | 路由                  | 状态    | 说明                                               |
| ---------------------------- | ---------- | --------------------- | ------- | -------------------------------------------------- |
| 抠图与背景 (Cutout & BG)     | 图片透明底 | `/background-removal` | ✅ 已有 | AI / 色键抠图，导出透明背景                        |
| 抠图与背景 (Cutout & BG)     | 证件照制作 | `/id-photo`           | 🆕 新增 | 抠图换底 + 标准尺寸裁剪 + 背景色                   |
| 转换与压缩 (Convert & Optim) | 格式转换   | `/convert`            | 🆕 新增 | PNG/JPG/WEBP 互转（高质量近无损）                  |
| 转换与压缩 (Convert & Optim) | 图片压缩   | `/compress`           | 🆕 新增 | 减小体积不损清晰度，可转最小体积格式、防增大       |
| 图片编辑 (Edit)              | 图片裁剪   | `/crop`               | 🆕 新增 | 框选矩形裁剪，比例约束，实时预览                   |
| 图片编辑 (Edit)              | 添加水印   | `/watermark`          | 🆕 新增 | 文字 / 图片(Logo) 水印，样式/位置/透明度，实时预览 |
| 视频工具 (Video)             | 视频截帧   | `/video-frames`       | ✅ 已有 | 精确 seek 取帧 + 帧动画预览                        |

### 3.1 整合决策与理由

- **「格式转换」与「压缩」合并为一个工具**：二者输入相同、参数高度重叠（目标格式 / 质量 / 输出尺寸），且用户常希望「转 WEBP 的同时压缩」，合并可减少跳转、提升效率。对应用户原始分类 ①。
- **「裁剪」独立为通用编辑工具**：裁剪是与具体场景无关的通用操作，独立后既可单独使用，也可被「证件照制作」复用。
- **「证件照制作」升级为独立场景化工具**：在用户原始分类 ②（裁剪 + 换底）基础上，整合「AI 抠图 + 背景色替换 + 标准证件照尺寸 + 裁剪定位」，形成完整的证件照解决方案，复用已有抠图能力。
- **「添加水印」独立编辑工具**：支持文字与图片(Logo)水印，与裁剪同属「图片编辑」分类，参数与预览逻辑独立。

## 4. 功能：图片透明底（背景移除）✅ 已有

路由：`/background-removal`

### 4.1 输入

- 支持拖拽 / 点击上传，可多选（批量）。
- 支持常见格式：PNG / JPG / JPEG / WEBP。

### 4.2 抠图模式（用户可选）

1. **AI 智能抠图**
   - 自动识别主体，去除任意背景。
   - 可选模型质量档位（速度优先 / 质量优先）。
2. **指定颜色抠图（色键 Chroma Key）**
   - 取色器选择要去除的背景颜色（或在预览图上吸管取色）。
   - 可调 **容差（Tolerance）**：颜色匹配范围。
   - 可调 **边缘羽化（Edge Feather）**：边缘平滑程度。

### 4.3 输出配置（用户可选）

- 导出格式：PNG（保留透明）/ WEBP（保留透明）/ JPG（不支持透明，用预览背景色填充，透明则填白）。
- 可选预览背景色（棋盘格或指定颜色）。
- 单张下载 / 批量打包 zip 下载。

### 4.4 交互

- 处理前后对比预览（棋盘格透明背景），原图/结果图均可点击放大。
- 批量任务列表，显示每张处理状态（等待 / 处理中 / 完成 / 失败）。
- **重复抠图**：单张可在调整参数后重新抠图。
- **结果勾选**：完成的结果可勾选，支持全选/全不选，仅打包下载勾选项。

## 5. 功能：视频截帧 ✅ 已有

路由：`/video-frames`

### 5.1 输入

- 支持拖拽 / 点击上传视频（MP4 / WEBM / MOV 等浏览器可解码格式）。

### 5.2 取帧方式（用户可选）

1. **固定时间间隔**：每隔 N 秒截取一帧（支持到 0.001s 的小间隔）。
2. **按帧率(FPS)**：填目标帧率，自动按 1/FPS 间隔取帧，适合还原丝滑动画。
3. **指定总帧数**：在视频时长内均匀截取 M 帧（M 可配）。
4. **手动截帧**：拖动进度条到任意时间点，点击按钮截取当前帧（追加，不清空）。
5. **时间区间**（可选增强）：限定开始/结束时间，再配合上面方式。

> 截帧原理：通过设置 `video.currentTime` 精确 seek 到目标时间点，等待 `seeked` 后用 canvas 抓取该帧，按时间精确取帧（非依赖播放）。

### 5.3 输出配置（用户可选）

- 导出格式：PNG / JPG / WEBP。
- **输出分辨率**：按比例缩放输出像素尺寸（原始～10%），影响清晰度与体积，不是质量压缩。
- JPG/WEBP 质量可调（独立的"图片质量"滑块，做有损压缩）。
- 单张下载 / 批量打包 zip 下载。

### 5.4 交互

- 视频预览播放器，一次只处理一个视频，重新上传按钮为"重新添加"（替换当前视频并清空旧帧）。
- 选择"固定间隔/帧率/指定帧数"后，实时预估输出张数；数量过大时给出耗时提醒，并显示提取进度。
- **批量提取会先清空上一次的结果**；手动截帧为追加。
- 截取的帧以缩略图网格展示，可单独删除、单独下载、全选打包，点击可放大。
- **帧动画预览**：将已截取的帧按可调帧率（FPS，1–60）播放一次（不循环），到末帧停止；可重新播放。

## 6. 功能：格式转换 与 图片压缩（两个独立路由）🆕 新增

路由：格式转换 `/convert`、图片压缩 `/compress`（拆为两个独立工具/路由，各司其职）。

> 对应用户原始分类 ①。同一处理管线：解码 → 缩放/质量调整 → 编码导出；两个路由共用结果列表 UI（上传、格式/体积对比、状态、批量下载、二次确认清空）。

### 6.1 输入

- 支持拖拽 / 点击上传，可多选（批量）。
- 支持常见格式：PNG / JPG / JPEG / WEBP（输入），统一通过 Canvas 解码处理。

### 6.2 图片压缩（`/compress`）

- **转为最小体积格式**（开关）：开启时自动在 WEBP / JPG / PNG 中编码并选体积最小者，采用高质量编码（**视觉无损，绝不降低分辨率**）。
- 关闭时：保持原格式压缩（PNG 为无损格式、压缩有限）+ 质量滑块（对有损格式生效）。
- **缩放尺寸**（可选开关）：按比例缩放，或限制最大宽/高（高质量重采样）。
- **防增大**：若压缩后体积不小于原图，自动**保留原图**，绝不输出更大文件。

### 6.3 格式转换（`/convert`）

- 目标格式：PNG / JPG / WEBP，仅做格式转换（高质量近无损，不缩放）。
- **背景填充色**：透明图转 JPG 时填充，默认白色。
- 提示：转换格式后体积可能变大（如 JPG→PNG）；只想减小体积请用「图片压缩」。

> 仅支持 PNG/JPG/WEBP 三种输出格式，不引入 HEIC / AVIF / GIF / BMP 等。

### 6.4 输出与反馈

- 实时显示**原始格式 → 输出格式**（格式徽标）、**原始体积 → 输出体积**与压缩率；体积增大时高亮警告。
- 单张下载 / 批量打包 zip 下载（按各自实际输出格式命名）。

### 6.5 交互

- 批量任务列表，清晰区分上传前后：格式徽标对比、体积对比、状态标签（待处理 / 处理中 / 已处理 / 失败）。
- 原图/结果图点击放大预览。
- 调整参数后可对全部重新处理。
- **「清空」仅清空处理结果，保留已上传图片**（危险样式 + 二次确认）；移除单张图片用每项的删除按钮。

## 7. 功能：图片裁剪 🆕 新增

路由：`/crop`

> 从用户原始分类 ② 拆分出的通用编辑能力。

### 7.1 输入

- 支持拖拽 / 点击上传单张图片（PNG / JPG / JPEG / WEBP）。

### 7.2 裁剪交互

- 点击上传后，在图上**拖拽绘制/调整矩形裁剪框**：四角手柄双向缩放；**四边手柄单轴缩放**（上/下仅垂直、左/右仅水平，自由比例下单轴微调），框内拖动移动位置。
- **比例约束**（用户可选）：自由比例、**原图片比例**（按当前图片宽高比锁定），以及参考市面产品（iLoveIMG / 美图 / Photoshop）的常用预设——1:1（头像）、3:4、4:3、2:3、3:2、5:7（证件照常用）、16:9、9:16，亦可自定义比例。
- **布局固定**：未上传图片时也保持左右布局——左侧裁剪菜单，右侧上传区。
- 可显示裁剪框的像素坐标与尺寸（x / y / 宽 / 高）。
- **实时预览**裁剪效果。

### 7.3 输出配置（用户可选）

- 导出格式：PNG / JPG / WEBP（JPG 透明填充背景色，默认白）。
- JPG/WEBP 质量可调。
- 下载裁剪结果。

### 7.4 交互

- 「重置」恢复初始裁剪框；「重新添加」替换当前图片。
- 原图/结果图点击放大预览。

## 8. 功能：证件照制作 🆕 新增

路由：`/id-photo`

> 在用户原始分类 ②（裁剪 + 背景替换成证件照）基础上整合升级，复用已有抠图能力。

### 8.1 输入

- 支持拖拽 / 点击上传单张人像图片（PNG / JPG / JPEG / WEBP）。

### 8.2 处理流程与参数（用户可选）

0. **抠图方式（用户可选，三选一）**：
   - **AI 智能抠图**（默认）：调用 AI 自动去除原背景（需联网下载模型，UI 提示），得到主体透明图后再换底。
   - **指定颜色替换背景（色键）**：指定原背景颜色（取色器）+ 容差 / 边缘羽化，将该颜色去除并替换为目标底色；适合纯色背景照片。参数调整时实时预览、保留裁剪框。
   - **不抠图（使用原图）**：跳过抠图，直接使用上传的图片。适用于**已是证件照/已抠好图**，只想**重新排版、裁剪或换底**的场景。
1. **背景色替换（目标底色）**：选择证件照标准背景色——白 / 蓝 / 红，或自定义颜色。
2. **标准尺寸**：内置常用规格（见下表），按所选规格输出像素尺寸；亦可自定义像素尺寸与 DPI（默认 300 DPI，部分场景如社保照可选 350 DPI）。
3. **裁剪定位**：在抠图结果上拖动/缩放裁剪框，将主体调整到证件照构图（复用裁剪能力），裁剪框按所选规格锁定宽高比。

#### 内置证件照规格表（300 DPI）

> 像素值按 300 DPI 由毫米换算（四舍五入），以市面常见取值为准；自定义规格时按 `像素 = 毫米 / 25.4 × DPI` 实时换算。

| 规格     | 毫米 (宽×高) | 像素 (宽×高, 300DPI) | 比例  | 常见用途           |
| -------- | ------------ | -------------------- | ----- | ------------------ |
| 一寸     | 25 × 35      | 295 × 413            | 5:7   | 简历、各类报名     |
| 二寸     | 35 × 49      | 413 × 579            | 5:7   | 简历、文凭         |
| 小一寸   | 22 × 32      | 260 × 378            | ~5:7  | 驾驶证             |
| 小二寸   | 35 × 45      | 413 × 531            | ~7:9  | 护照（部分）、签证 |
| 大一寸   | 33 × 48      | 390 × 567            | ~7:10 | 护照、港澳通行证   |
| 大二寸   | 35 × 53      | 413 × 626            | ~2:3  | 简历、毕业证       |
| 身份证   | 26 × 32      | 358 × 441            | ~9:11 | 第二代居民身份证   |
| 护照     | 33 × 48      | 390 × 567            | ~7:10 | 中国护照           |
| 中国签证 | 33 × 48      | 354 × 472            | ~3:4  | 中国签证（白底）   |

> 背景色规范：护照、签证、社保等官方证件多要求**白底**；蓝 / 红底多用于其他场景。

### 8.3 输出配置（用户可选）

- 导出格式：PNG / JPG（默认 JPG，证件照常用）。
- 质量可调。
- 下载单张结果。
- **排版打印（必做）**：将多张同一张证件照按所选规格拼版到标准相纸（默认 6 寸 4R = 102×152mm，亦可选 5 寸等），自动按相纸尺寸最大化排布行列、保留裁切间距与可选裁切线，导出可直接送冲印的拼版大图。

### 8.4 交互

- 步骤式引导（上传 → 抠图换底 → 选尺寸/背景 → 裁剪定位 → 导出），各步骤实时预览。
- 联网提示：首次需下载抠图模型。
- 原图/结果图点击放大预览。

## 9. 功能：添加水印（文字 / 图片）🆕 新增

路由：`/watermark`

> 对应用户原始分类 ③，并扩展支持图片（Logo）水印。

### 9.1 输入

- 支持拖拽 / 点击上传，可多选（批量，统一应用同一水印配置）。
- 支持常见格式：PNG / JPG / JPEG / WEBP。

### 9.2 水印参数（用户可选）

- **水印类型**：**文字水印** 或 **图片水印（Logo）**，可单独或同时使用。
- **文字水印**：
  - 水印文字：用户输入文字内容。
  - 字体：可选字体族（提供若干内置可用字体）。
  - 字号：可调字号（或按图片宽度比例自适应）。
  - 颜色：取色器选择文字颜色。
- **图片水印（Logo）**：
  - 上传 Logo 图片（PNG 带透明优先）。
  - 缩放：按图片宽度比例或像素调整 Logo 大小。
- **文字 + 图片（多布局）**：选择「文字 + 图片」时，支持文字与 Logo 的相对布局与对齐——**图上 / 图下 / 图左 / 图右**，文字与图片在各自单元内自动居中对齐。
- **通用样式**（对文字 / 图片水印均生效）：
  - 透明度：0–100% 可调。
  - 位置：九宫格定位（左上 / 上中 / 右上 / … / 右下）或自定义坐标拖动。
  - 旋转角度：**0–360° 可调**。
  - 平铺模式（可选）：整图重复平铺水印，可调间距，用于防盗用。
  - （可选增强）文字描边/阴影以增强不同底色上的可读性。

### 9.3 输出配置（用户可选）

- 导出格式：PNG / JPG / WEBP。
- JPG/WEBP 质量可调。
- 单张下载 / 批量打包 zip 下载。
- **「清空」仅清空处理结果，保留已上传图片**；移除单张图片用每项的删除按钮。

### 9.4 交互

- **实时预览**水印效果（在画布上叠加）。
- 批量列表展示每张处理状态，支持单张下载与打包下载。
- 原图/结果图点击放大预览。

## 12. 需求确认结论（已确认）

> 以下产品决策已与需求方确认，作为实现依据。

1. **格式转换与压缩**：仅支持 PNG / JPG / WEBP，不引入 HEIC / AVIF / GIF / BMP 等其他格式。
2. **证件照规格**：内置一寸、二寸、小一寸、小二寸、大一寸、大二寸、身份证、护照、中国签证等常用规格（见 §8.2 规格表），支持自定义尺寸与 DPI；**需要排版打印**（拼版到 6 寸等标准相纸，可直接送冲印，见 §8.3）。
3. **裁剪比例**：提供市面常用预设——自由、1:1、3:4、4:3、2:3、3:2、5:7、16:9、9:16，并支持自定义比例（见 §7.2）。
4. **水印**：**需要图片(Logo)水印**，与文字水印并存，支持透明度 / 位置 / 旋转 / 平铺等通用样式（见 §9.2）。
5. **导航形态**：采用**左侧分组侧边栏**，按分类分组展示各功能（见 §3、§10）。

### 12.1 第二轮细化确认

6. **证件照抠图方式**：用户可选 **AI 智能抠图** 或 **不抠图（使用原图）**；已是证件照/已抠图、仅需排版或换底时无需 AI 抠图（见 §8.2）。证件照排版打印为必做项。
7. **格式转换与压缩解耦**：转换与压缩为相互独立的选项；新增「最小体积（自动）」目标格式（体积优先，自动选最小格式）；压缩采用高质量重采样保证清晰度；明确提示「转换格式后体积可能变大」并在结果中高亮（见 §6.2 / §6.3）。
8. **转换结果对比清晰化**：结果行清晰展示 原始格式→输出格式、原始体积→输出体积、状态标签（待处理 / 处理中 / 已处理 / 失败）。
9. **裁剪比例**：新增「**原图片比例**」预设（见 §7.2）。
10. **水印旋转与图文布局**：旋转角度支持 **0–360°**；「文字 + 图片」支持图上/图下/图左/图右 多布局对齐（见 §9.2）。
11. **「清空」语义**：批量工具（格式转换、添加水印）的「清空」**仅清空处理结果，保留已上传图片**；移除单张用每项删除按钮（见 §6.4 / §9.3）。

### 12.2 第三轮细化确认

12. **证件照抠图方式（三选一）**：AI 智能抠图 / **指定颜色替换背景（色键：选原背景色 + 容差/羽化，替换为目标底色）** / 不抠图（用原图）。色键参数调整时实时重处理并保留裁剪框（见 §8.2）。
13. **证件照布局固定**：无论是否已上传图片，始终采用**左侧菜单 + 右侧工作区**的左右布局；未上传时右侧为上传区。
14. **格式转换与压缩拆为两个板块**（Tabs）：
    - **压缩**：可勾选「转为最小体积格式」（自动选 WEBP/JPG/PNG 最小者，高质量编码、**视觉无损**）；或保持原格式压缩 + 可选缩放。**若压缩后体积反而更大，自动保留原图，绝不输出更大文件**。
    - **转换格式**：仅做格式转换（高质量近无损）；提示「转格式可能增大体积，减小体积请用压缩板块」。
15. **裁剪自由比例单轴拖动**：裁剪框新增**四边手柄**——上/下仅垂直拖动、左/右仅水平拖动（自由比例下单轴调整）；四角仍为双向。
16. **裁剪布局固定**：未上传图片时也保持左右布局，左侧为裁剪菜单，右侧为上传区。
17. **清空结果按钮**：保留 `destructive` 危险样式，并增加**二次确认弹窗**（格式转换、添加水印）。

### 12.3 第四轮细化确认

18. **格式转换与图片压缩拆为两个独立路由**：`/convert`（格式转换）与 `/compress`（图片压缩）分别为独立工具与导航入口，不再用 Tab 合并；两者共用同一套结果列表 UI（上传、格式/体积对比、状态、批量下载、二次确认清空）。
